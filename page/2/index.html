Vue 介绍

渐进式框架：可以用最简单的方式直接导入 vue.js 使用，随着项目变大，可以逐步集成周边插件。

  • 版本：Vue 3
  • 学习方式：选项式 API
  • 官网：https://cn.vuejs.org/
  • CDN 地址：https://unpkg.com/vue@3/dist/vue.global.js

  1. 下载 vue.js 文件
  2. 在 html 中引入
  3. 创建 Vue 实例
  4. 挂载到页面元素

动态属性：:[attr]="url" 可以动态绑定不同属性

响应式原理

  1. 虚拟 DOM 本质是一个 JS 对象
  2. 数据改变先更新虚拟 DOM，不直接操作真实 DOM
  3. 通过 DIFF 算法 比较新旧虚拟 DOM，只更新变化的部分
  4. nextTick 在真实 DOM 更新后调用

data 必须是函数返回对象：保证每个组件有独立的数据作用域，通过 Proxy 实现响应式

计算属性 (computed)

特点

  1. ✅ 有缓存
  2. ✅ 立即执行一次
  3. ✅ 监听多个依赖，任一改变就重新计算
  4. ✅ 必须有 return 返回值
  5. ✅ 多个值影响一个值

可写计算属性

侦听器 (watch)

特点

  1. ✅ 惰性：默认不立即执行
  2. ✅ 无需返回值
  3. ✅ 支持异步操作
  4. ✅ 没有缓存
  5. ✅ 监听一个值，影响多个值

watch vs computed 对比

特性 computed watch
立即执行 ✅ 是 ❌ 否（惰性）
返回值 ✅ 需要 ❌ 不需要
异步 ❌ 不支持 ✅ 支持
缓存 ✅ 有 ❌ 无
监听数量 多个值 一个值
场景 多对一 一对多

Class 和 Style 绑定

Class 绑定

  • 字符串：:class="className"
  • 对象：:class="{ active: isActive }" 布尔值控制
  • 数组：:class="[classA, classB]"

绑定的 class 会和原有的 class 合并

Style 绑定

同样支持字符串、对象、数组三种方式

条件渲染

v-if / v-else-if / v-else

  • 条件成立才渲染，不成立直接移除
  • 具有惰性
  • 适合切换不频繁的场景

v-show

  • 始终渲染，通过 CSS display 控制显示隐藏
  • 非惰性
  • 适合频繁切换的场景

列表渲染 v-for

可遍历类型

  • 数字：v-for="num in 10" 从 1 开始
  • 字符串：v-for="str in '你好'" 逐个字符
  • 数组：v-for="(item, index) in arr"
  • 对象：v-for="(value, key, index) in obj"

key 属性

使用 v-for 必须加 key，且唯一。帮助虚拟 DOM 正确识别元素位置，提升 diff 性能。

v-if 和 v-for 优先级

  • Vue 2：for 优先级高于 if
  • Vue 3：if 优先级高于 for（条件不成立不循环）

不推荐一起使用

Git 使用教程

发表于 2026-07-16 分类于 开发工具

Git 代码管理及控制工具

目前最常用的代码管理工具: git 和 svn

对于程序员而言，最重要的是代码

对于公司而言，花大笔钱来养的不是程序员，而是他们的代码

在公司中开发一个项目可能有多个前端工程师，但是每天写完代码咱们要将所有代码整合在一起

Git 的使用

  • 第一步: 下载安装 https://git-scm.com/ 安装路径没有任何要求

  • 第二步: git init 初始化一个 git 仓库（本地仓库）。创建一个文件夹 my-git，创建一个文件 xyj.html

  • 第三步: git add xyj.html

追踪、跟踪文件代码，注意后缀名要加上，将创建好的项目提交到缓存区

  • 第四步: git commit -m '提交描述'

将我们刚刚追踪的文件提交到本地仓库中，-m 是提交时的描述信息

我们需要自己配置邮箱（该邮箱可以不存在），此时是第一次提交，git 会提示你没有邮箱和密码：

配置好邮箱和用户名之后，再次提交即可

  • 第五步: 查看提交记录

可以查看到当前的版本信息，也可以看到我们的版本 id 和提交人的信息，还有提交时间和描述信息

开始编写代码

  • 编写第一天代码，其实就是我们一天的代码写完了之后
  • 当写完第二天的代码后，执行 git add . 追踪，执行 git commit -m '第二回完'
  • 第三天写完代码后可以通过 git diff 来查看当前代码和上一次提交的代码有什么区别
  • 当写完第三天的代码后，执行 git add . 追踪，执行 git commit -m '第三天提交'

将代码回退到某个版本

  • 只回退代码的版本信息，不回退代码

分支

（在自己的分支中写东西，不能去别人的分支写东西）

带有 * 号的分支表示当前所在的分支，master 是我们的主分支

cxq 写完第四回，然后提交第四回

将分支切换到 master 分支后，并没有 cxq 分支中的代码

因为我们分支切换（git checkout master）到 master 分支后，并没有将 cxq 分支的代码合并过来，所以我们主分支中就没有看到 cxq 的代码

假设有 a、b、c 三个人都在写代码，下班前 a、b、c 三人都将自己的代码进行提交，提交完成之后由主分支（master）一一合并

远程仓库的操作

  1. 在远程仓库中新建一个仓库
  2. 将远程仓库克隆到本地：git clone https://gitee.com/jzl3536/myxyj2218.git
  3. 如果是一个私有项目，clone 时候需要账号密码；如果是一个开源项目，clone 时候不需要账号密码
  4. 开始写代码，创建一个项目，进入项目安装 node 包，然后将项目跑起来
  5. cd 到有 .git 文件的目录下，执行命令 git add . 先追踪所有文件
  6. 提交文件：git commit -m '项目创建成功'，这里提交的是提交到本地
  7. 执行命令 git push 将本地的最新代码提交到远程仓库

每天写代码之前先 git pull

先执行命令 git pull 将远程仓库中最新的代码拉下来
