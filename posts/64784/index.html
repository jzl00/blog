React Hooks 完整教程

发表于 2026-07-16 分类于 前端开发 阅读次数：

useState

用来在函数式组件中定义状态。

示例：

useReducer

可以替代 useState 创建状态，适合管理大量状态或复杂状态逻辑。

useContext

接收祖先组件通过 Context.Provider 传递的数据。

useRef

创建 ref 属性，用来标记 DOM 元素或组件实例。

htmlFor 代替 for 属性，用来和 input 绑定

useImperativeHandle

配合 forwardRef 使用，自定义暴露给父组件的内容。

useEffect

替代类组件的 componentDidMount 和 componentDidUpdate。

  • 第一个参数：回调函数，挂载和更新时执行
  • 第二个参数：依赖数组，只有依赖改变才执行
  • **空数组 []**：只在挂载时执行一次

常见用法：

useLayoutEffect

与 componentDidMount、componentDidUpdate 调用阶段相同，同步执行。

  • useEffect — 异步执行
  • useLayoutEffect — 同步执行，在 useEffect 之前调用

推荐先用 useEffect，出问题再考虑 useLayoutEffect

useInsertionEffect

也是同步函数，执行顺序在 useLayoutEffect 和 useEffect 之前。

主要用于 CSS-in-JS 库注入样式。

useCallback

用来缓存函数，避免子组件不必要的重新渲染。

使用场景： 子组件使用了父组件的方法，父组件其他状态变化导致重新渲染时，子组件也会被重新渲染。用 useCallback 包裹函数可以缓存函数引用。

useMemo

用来缓存计算结果，针对状态优化性能。

使用场景： 子组件接收的数据是由多个属性组合而成的对象（如 const info = { name, age }），父组件重新渲染会导致对象重新创建。用 useMemo 包裹可以避免不必要的重新计算。

useCallback vs useMemo

Hook 缓存内容 返回值
useCallback 函数 函数
useMemo 计算结果 值

useCallback(fn, deps) 相当于 useMemo(() => fn, deps)

useTransition

定义修改数据时的非阻塞机制。

  • isPending — 布尔值，是否处于等待状态
  • startTransition — 开启过渡的方法

用于将更新标记为非紧急更新，不阻塞 UI 响应。

useDeferredValue

传递一个值，返回该值的延迟副本。

用于延迟更新非紧急部分的 UI，类似防抖效果。

useDebugValue

用来标记自定义 Hook，可以在 React 开发工具中查看。

useId

生成唯一 ID，可用于无障碍属性等。

不能作为 CSS 选择器，也不能用来获取 DOM

useSyncExternalStore

允许组件订阅外部存储（如 Redux 等状态管理库）。

Hooks 使用规则

  1. 只在最顶层使用 Hook — 不要在循环、条件或嵌套函数中调用
  2. 只在 React 函数中调用 — React 函数组件或自定义 Hook
  3. 自定义 Hook 以 use 开头
